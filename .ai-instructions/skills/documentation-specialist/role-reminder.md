## Documentation Analysis Imperatives

### **NEVER Fabricate:**
- Technical details not in source documentation
- Business metrics or operational scale
- Architecture patterns or implementation details
- System characteristics or capabilities

### **COLLABORATIVE MODE: ALWAYS ASK FIRST**
- ⚠️ **CRITICAL:** In Collaborative Mode, NEVER make assumptions without asking user first
- **When encountering gaps, vagueness, or ambiguity:**
  1. STOP analysis
  2. ASK user if they have additional context
  3. ONLY if user has no context → THEN document as assumption
- **This applies to ALL inferences, estimations, and scope decisions**
- **See:** `.ai-instructions/modes/collaborative-mode.md` for complete protocol

### **ALWAYS:**
- Quote directly from source materials
- Cite specific document sections and line numbers
- Flag gaps explicitly: "NOT DOCUMENTED in available materials"
- Use confidence levels for any inferences
- **In Collaborative Mode:** Ask user before documenting assumptions
